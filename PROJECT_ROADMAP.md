# Deception Archive - Initial Roadmap

This document captures the agreed approach for building the first version of the Deception Archive (Library of Lies).

## Product Direction

- Start frontend-first with mock data.
- Build in very small steps with frequent review.
- Prioritize a cool, interactive, and original user experience.
- Treat current ideas as guidance, not rigid requirements.

## Why this approach

- Fast visual prototype for team feedback.
- Low-risk iteration before backend commitments.
- Clear separation between structure, design, and data integration.

## Visual References

- Primary style reference: https://projectjustice.eu
- Current color direction: "Faded Testimony" (aged parchment, foxed brown, muted ink-green).

## Build Order

1. Step 1 - Basic project scaffold (frontend only)
2. Step 2 - Design foundation (typography, color tokens, spacing)
3. Step 3 - Homepage skeleton (hero + contribution CTA + dataset grid layout)
4. Step 4 - Reusable dataset cards with key metadata
5. Step 5 - Search, filters, sorting, and basic interactions
6. Step 6 - Dataset detail page
7. Step 7 - Glossary/metadata definitions page
8. Step 8 - Meta-dataset explorer (combined dataset overview)

## Candidate Metadata for Dataset Cards

- Dataset name
- Short description
- Language
- Number of statements
- Ground truth procedure
- Topic of deception
- Experimental design
- Domain/topic tags
- Citation/source links

## Immediate Next Action

Initialize a clean frontend scaffold and keep implementation intentionally minimal. We will then agree on the visual direction before adding feature logic.
